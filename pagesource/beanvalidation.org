          <h2 id="what-is-bean-validation"><a class="anchor" href="#what-is-bean-validation"></a>What is Bean Validation</h2>
          <div class="sectionbody">
          <div class="paragraph">
          <p>Bean Validation is a Java specification which</p>
          </div>
          <div class="ulist">
          <ul>
          <li>
          <p>lets you express constraints on object models via annotations</p>
          </li>
          <li>
          <p>lets you write custom constraints in an extensible way</p>
          </li>
          <li>
          <p>provides the APIs to validate objects and object graphs</p>
          </li>
          <li>
          <p>provides the APIs to validate parameters and return values of methods
          and constructors</p>
          </li>
          <li>
          <p>reports the set of violations (localized)</p>
          </li>
          <li>
          <p>runs on Java SE but is integrated in Java EE 6 and later; Bean Validation 2.0 is part of Java EE 8</p>
          </li>
          </ul>
          </div>
          <div class="listingblock">
          <div class="content">
          <pre class="CodeRay highlight"><code data-lang="java"><span style="color:#088;font-weight:bold">public</span> <span style="color:#339;font-weight:bold">class</span> <span style="color:#B06;font-weight:bold">User</span> {&#x000A;&#x000A;    <span style="color:#088;font-weight:bold">private</span> <span style="color:#0a8;font-weight:bold">String</span> email;&#x000A;&#x000A;    <span style="color:#007">@NotNull</span> <span style="color:#007">@Email</span>&#x000A;    <span style="color:#088;font-weight:bold">public</span> <span style="color:#0a8;font-weight:bold">String</span> getEmail() {&#x000A;      <span style="color:#080;font-weight:bold">return</span> email;&#x000A;    }&#x000A;&#x000A;    <span style="color:#088;font-weight:bold">public</span> <span style="color:#339;font-weight:bold">void</span> setEmail(<span style="color:#0a8;font-weight:bold">String</span> email) {&#x000A;      <span style="color:#950">this</span>.email = email;&#x000A;    }&#x000A;}&#x000A;&#x000A;<span style="color:#088;font-weight:bold">public</span> <span style="color:#339;font-weight:bold">class</span> <span style="color:#B06;font-weight:bold">UserService</span> {&#x000A;&#x000A;  <span style="color:#088;font-weight:bold">public</span> <span style="color:#339;font-weight:bold">void</span> createUser(<span style="color:#007">@Email</span> <span style="color:#0a8;font-weight:bold">String</span> email,&#x000A;                         <span style="color:#007">@NotNull</span> <span style="color:#0a8;font-weight:bold">String</span> name) {&#x000A;    ...&#x000A;  }&#x000A;}</code></pre>
          </div>
          </div>
          <div class="paragraph">
          <p>While you can run validation manually, it is more natural to
          let other specifications and frameworks validate data at the right
          time (user input in presentation frameworks, business service
          execution by CDI, entity insert or update by JPA).</p>
          </div>
          <div class="paragraph">
          <p>In other words, <em>run once, constrain anywhere</em>.</p>
          </div>
